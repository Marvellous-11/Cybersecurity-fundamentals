# 🛡️ Asset Security & Risk Management

This project documents my learning and practical understanding of asset security, risk management, and cybersecurity frameworks.

---

## 🔹 Overview

Asset security involves identifying, classifying, and protecting both digital and physical assets within a system. This includes understanding risks, threats, and vulnerabilities that can impact these assets.

---

## 🔹 Key Concepts Learned

### 1. Assets in Cybersecurity
- **Digital Assets:** Data, systems, networks, and applications  
- **Physical Assets:** Devices such as computers, routers, and storage systems  

---

### 2. Asset Classification
- Categorizing assets based on sensitivity and importance  
- Common classification levels:
  - Public  
  - Internal  
  - Confidential  
  - Restricted  

---

### 3. States of Data
- **Data at Rest** – stored data  
- **Data in Transit** – data being transferred  
- **Data in Use** – data actively being processed  

---

### 4. Risks, Threats, and Vulnerabilities
- **Risk:** Potential for loss or damage  
- **Threat:** Possible danger to an asset (e.g. malware, phishing)  
- **Vulnerability:** Weakness that can be exploited  

---

### 5. Security Planning
Key elements of a security plan:
- Asset identification  
- Risk assessment  
- Implementation of controls  
- Continuous monitoring  

---

### 6. NIST Cybersecurity Framework
The NIST framework provides guidelines for managing cybersecurity risks:

- Identify  
- Protect  
- Detect  
- Respond  
- Recover  

---

### 7. Cloud Security
- Protection of data stored in cloud environments  
- Importance of access control and data encryption
